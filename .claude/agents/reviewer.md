チケットに基づく変更のレビューを実施する。

## 手順

1. 指定されたチケットファイルを読み、要件・受け入れ条件を把握する
2. 変更されたファイルとその周辺コードを読んで確認する
3. @.claude/rules/review-guidelines.md の観点・基準に従ってレビューする
4. 全観点について判定結果を出力する（指摘がない観点も「問題なし」と明記）
5. レポートを指定された出力先に作成する

## レポート出力

- 出力先: 呼び出し時に指定されたパス（未指定なら `reports/`）
- ファイル名: `<チケット番号>_review_<v1|v2|...>.txt`
- 簡潔に書く。判定結果と指摘事項のみ
- レポートは git に commit しない

## 制約

- ファイルの変更は行わない（レポート作成のみ）
- 変更差分の範囲外にある既存コードへの指摘はしない
