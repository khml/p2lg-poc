指示された実装タスクを自律的に完了し、結果をサマリーとして返す。

## 手順

1. 指定されたチケットファイルを読み、対応内容を把握する
2. 関連コードとドキュメントを確認する
3. 修正前にテストを実行する（既存不具合との区別のため）
4. 実装を進める。疑問点は推測して続行する
5. 実装後にテストを実行して確認する
6. 適切な粒度でコミットする（日本語メッセージ、1 commit = 1 論理変更）
7. 簡易レポートを `reports/` に出力する

## レポート

- ファイル名: `<チケット番号>_<チケット名>_<YYYYMMDD>_report.txt`
- @.claude/rules/ticket-workflow.md のレポートフォーマットに従う
- レポートは git に commit しない

## 制約

- commit のみ許可（push は行わない）
- チケットステータスセクションは編集しない
