# チケット作成

## 手順

1. `tickets/.latest_ticket_number` を読み、現在の最新番号を確認する
2. 次の番号を採番する（+1）
3. 以下のフォーマットでチケットファイルを作成する
4. `tickets/.latest_ticket_number` を新しい番号で更新する
5. チケットファイルと `.latest_ticket_number` を commit する
   - commit メッセージ: `チケット追加: <番号> <タイトル>`
   - 複数チケット一括作成の場合: `チケット追加: <番号1>, <番号2> <タイトルを列挙>`

## フォーマット

ファイル名: `tickets/<4桁番号>_<チケット名>.txt`

```
チケット番号: <4桁番号>
タイトル: <チケットタイトル>

---

## 概要
<このチケットで何を行うかの概要>

## 背景
<なぜこのチケットが必要なのか>

## 対応内容
- <具体的な対応項目>

## 優先度
<高/中/低>

## 依存関係
- <依存するチケットがあれば記載>

---

## チケットステータス
未着手
```

## 複数チケットの一括作成

ユーザーから複数のチケット作成を依頼された場合は、連番で採番し、最後に `.latest_ticket_number` を最終番号で更新する。commit は一括で行う。

## ファイルの移動・リネーム

チケットファイルを移動またはリネームする際（done/ への移動、ファイル名変更など）は、
Write ツールではなく `git mv <旧パス> <新パス>` を使う。
Write で再作成すると git が削除＋新規追加として扱い、履歴の連続性が失われる。
