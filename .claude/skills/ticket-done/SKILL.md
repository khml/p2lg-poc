# チケット完了処理

## 手順

1. 対象チケットファイルを `tickets/done/` に移動する（**必ず `git mv` を使う**）
2. `docs/roadmap.md` を読み、更新箇所を特定する
3. ロードマップを更新する
4. まとめてコミットする

---

## ステップ 1: チケット移動

```bash
git mv tickets/<番号>_<タイトル>.txt tickets/done/<番号>_<タイトル>.txt
```

> **注意**: Write ツールで再作成してはいけない。`git mv` でないと履歴が断絶する。

---

## ステップ 2 & 3: ロードマップ更新

`docs/roadmap.md` を読んで以下の箇所を更新する:

### 現在のバージョン・状態セクション

- `最近完了したフェーズ` を更新（該当する場合）
- `完了済みフェーズ数` を更新（該当する場合）

### 現在の状態サマリーセクション

完了内容を箇条書きで追記する。形式:

```
- **<タイトル>（<番号> 完了）** （<YYYY-MM-DD>）
  - <実施内容を箇条書き>
```

### 未着手チケット一覧セクション

該当チケットの行の `ステータス` を `未着手` → `完了` に変更する。

チケットが既存フェーズに属さない場合は、新フェーズのテーブルを「未着手チケット一覧」の末尾に追加する:

```markdown
### Phase XX: <フェーズ名>

| チケット番号 | タイトル | フェーズ | 優先度 | ステータス |
|---|---|---|---|---|
| XXXX | <タイトル> | Phase XX | <優先度> | 完了 |
```

### 今後の開発方向性セクション

新フェーズを追加した場合はここにも追記する。

---

## ステップ 4: コミット

```bash
git add tickets/done/<番号>_<タイトル>.txt docs/roadmap.md
git commit -m "チケット移動・ロードマップ更新: <番号> 完了"
```

複数チケットを一括処理する場合:
```bash
git commit -m "チケット移動・ロードマップ更新: <番号1>, <番号2> 完了"
```
