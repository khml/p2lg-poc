# レビューワークフロー

## フェーズ 1: セルフレビュー

reviewer に渡す前に、自分でセルフレビューを行う。

- @.claude/rules/review-guidelines.md の観点に基づいて自己チェック
- 問題を発見した場合は修正してからレビューに進む

## フェーズ 2: reviewer サブエージェントの起動

reviewer サブエージェントを起動し、以下の情報を渡す:

- 対象チケットのファイルパス
- レビュー対象のコミットハッシュ
- レポート出力先（`reports/<チケット番号>_review_v1.txt`）

## フェーズ 3: レビュー結果の確認

reviewer からレポートを受け取り、結果をユーザーに報告する。

- **✅ Approve**: 完了。ユーザーに報告する
- **⚠️ Approve with comments**: Warning の内容をユーザーに報告し、対応要否を確認する
- **❌ Request Changes**: フェーズ 4 に進む

## フェーズ 4: 指摘対応

Critical と Warning の指摘に対して修正を実施する。

- 修正を commit する
- 修正内容を簡潔にまとめる

## フェーズ 5: 再レビュー

reviewer を再度起動する（レポートのバージョンをインクリメント: `v2`, `v3`, ...）。
Approve が得られるまでフェーズ 3〜5 を繰り返す。

3 回繰り返しても Approve にならない場合は、ユーザーに状況を報告し判断を仰ぐ。
